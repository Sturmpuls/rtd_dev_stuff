Verhalten von Constraints¶

PRIMARY KEY entspricht in seinem Verhalten fast gänzlich der Kombination aus UNIQUE und NOT NULL. Für manche Anwendungsfälle, z.B. wenn Daten mit ogr2ogr importiert werden sollen, kann es nötig sein letzteres zu verwenden. Der einzige Vorteil von PRIMARY KEY ist, dass Postgres automatisch auf ihn zurück greift falls ein FOREIGN KEY sich auf die Tabelle bezieht ohne eine Spalte anzugeben.
